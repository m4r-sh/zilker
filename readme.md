# zilker

Parametric Hub

Dev environment centered around: Inputs -> Outputs

Creativity = (Intuitive Input) x (Specific Output)

-> Most intuitive input tools limit output specificity
-> Most specific output tools are arcane and unwiedly

A project architect should configure specific output patterns and prune the inputs to the intuitive essentials. This gives operators the freedom to create without friction, generating the files they need without any added steps or time.


Instant feedback loop is essential. (Iteration Speed)
Simple code is essential. (LLM gen)
Complex interdependent file needs are inevitible.

-> Parametric Editing is going nowhere
-> JavaScript / Bun is king for LLMs and ubiquity and speed
-> The web is just getting started


This tool is starting simple, but eventually:
- Can be optimized for raw speed once the API is solid
- Can have granular dev view in the browser
- Can manage version control and deployment tasks
- Can accomodate LLM contributions
- Can accept json / sql data eiher fetched or local as inputs
- Can run natively within the browser (decouple from native apis)
- Can integrate with terminal for quick scaffolding, prompting, or mirror whatever the browser viewer can do
